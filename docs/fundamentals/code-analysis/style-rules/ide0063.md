# Use simple 'using' statement (IDE0063)

| Property                 | Value                                   |
|--------------------------|-----------------------------------------|
| **Rule ID**              | IDE0063                                 |
| **Title**                | Use simple `using` statement            |
| **Category**             | Style                                   |
| **Subcategory**          | Language rules (code block preferences) |
| **Applicable languages** | C# 8.0+                                 |

## Overview

This style rule concerns the use of `using` statements without curly braces. This alternative syntax was introduced in C# 8.0.

## csharp_prefer_simple_using_statement

| Property                 | Value                                                                                                                     |
|--------------------------|---------------------------------------------------------------------------------------------------------------------------|
| **Option name**          | csharp_prefer_simple_using_statement                                                                                      |
| **Option values**        | `true` - Prefer to use a *simple* `using` statement<br /><br />`false` - Don't prefer to use a *simple* `using` statement |
| **Default option value** | `true`                                                                                                                    |

### Example

```csharp
// csharp_prefer_simple_using_statement = true
using var a = b;

// csharp_prefer_simple_using_statement = false
using (var a = b) { }
```
### is using the new syntax a must?
actually using "`using` new syntax, tells the compiler that the variable being declared should be disposed at the end of the enclosing scope" so if you have nothing after `using statement` but a closing curly bracket, IDE will suggest you to use new syntax otherwise you should think about it that if there is a difference between using new syntax or not (which in over 99% of times it won't)
```csharp
// IDE throw suggestions/warnings for the following code:
// in the following code you should use new syntax cause the only difference between new and old syntax is more typing in old one:
private void CreateContact()
{
    Contact contact = new Contact()
    {
        Name = name.Text,
        Family = family.Text
    };
    using SQLiteConnection conn = new(App.DbPath);
    conn.CreateTable<Contact>();
    try
    {
        conn.Insert(contact);
        Message.Success();
        ClearFields();
    }
    catch (Exception ex)
    {
        Message.Failed(ex);
    }
}
// no IDE suggestions/warnings example:
// in the following code you can still use new syntax cause it will close the connection after method execution but there is one difference between new and old syntax, in new syntax variable will Dispose at the end of the method but in the old syntax the variable will dispose before the `if`:
private void ReadContact()
{
    using(SQLiteConnection conn = new(App.DbPath))
    {
        conn.CreateTable<Contact>();
        contacts = conn.Table<Contact>().ToList();
    }
    if (contacts.Count > 0)
    {
        contactsList.ItemsSource = contacts;
    }
}
```
